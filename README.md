# Tugas3_JS

Berbeda dengan skrip sederhana yang hanya menerima input mentah, di program ini saya menggunakan Node.js dengan pendekatan yang lebih interaktif. Di bagian awal, saya mengimpor modul readline untuk menciptakan antarmuka tanya-jawab di terminal. Jadi, program tidak langsung berjalan begitu saja, tapi dia akan menyapa pengguna dan meminta input tanggal serta bulan lahir secara berurutan menggunakan fungsi rl.question.

Bagian yang paling saya perhatikan dalam kode ini adalah sistem validasinya. Saya tidak ingin program ini menghasilkan data yang ngawur kalau penggunanya iseng. Makanya, sebelum masuk ke logika zodiak, saya menyisipkan pengecekan bulan agar tetap berada di rentang 1 sampai 12. Untuk tanggalnya pun saya buat lebih cerdas; saya menggunakan trik new Date(2024, bln, 0).getDate() untuk mengecek berapa jumlah hari maksimal di bulan tersebut. Dengan cara ini, kalau ada yang memasukkan tanggal 30 di bulan Februari, program saya akan langsung menolak karena dia tahu batas maksimalnya tidak sampai sana.

Untuk penentuan zodiaknya sendiri, saya tidak menggunakan tabel statis, melainkan rangkaian logika pengkondisian if-else yang sangat spesifik. Karena setiap zodiak biasanya "menyeberang" di antara dua bulan, saya menggabungkan kondisinya menggunakan operator logika OR (||). Sebagai contoh, untuk menentukan Aries, saya buat aturan: jika lahir di bulan 3 (Maret) mulai tanggal 21, ATAU lahir di bulan 4 (April) hingga tanggal 19. Logika ini saya susun secara runtut dari Januari sampai Desember agar tidak ada satu tanggal pun yang terlewat.

Terakhir, setelah semua data tervalidasi dan zodiaknya ditemukan, program akan menampilkan ringkasan hasilnya di layar terminal. Sebagai penutup yang rapi, saya menggunakan fungsi rl.close() agar proses input-output di terminal berhenti secara otomatis setelah tugasnya selesai. Secara keseluruhan, kode ini bukan cuma sekadar penebak zodiak, tapi sebuah sistem yang sudah saya proteksi dengan logika error handling yang cukup matang.
